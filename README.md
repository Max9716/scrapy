Проект представляет собой промышленный парсер интернет-магазина Алкотека, реализованный на фреймворке Scrapy в рамках тестового задания.

Решение демонстрирует умение работать с внутренним API сайта, структурировать данные по заданному шаблону и учитывать региональные особенности отображения информации.

Что реализовано:

- Прием входного списка категорий (минимум 3 ссылки)
- Обязательный учет региона — Краснодар
- Сбор данных через API (без использования Selenium/Playwright)
- Полное соответствие формату выходных данных из ТЗ
- Корректный расчет процента скидки
- Сбор наличия и остатков товара
- Обработка маркетинговых тегов
= Сбор изображений и медиаданных
- Добавление всех характеристик товара в metadata
- Поддержка работы через прокси
= Заполнение обязательных полей даже при отсутствии данных

Технические особенности:

- Используется внутренняя структура API сайта
- Безопасная обработка отсутствующих данных
= Четкая структура выходного JSON
= Масштабируемая архитектура (легко расширяется под pagination и middleware)
- Готовность к промышленному использованию

▶ Запуск:
scrapy runspider  alcoteka.py -O alcoteka.json -s FEED_EXPORT_ENCODING=utf-8

Результатом выполнения является файл result.json со структурированными данными товаров по заданным категориям.
